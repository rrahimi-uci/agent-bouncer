# Archived broad-study Benchmark Explorer

This self-contained page belongs to the earlier broad study. Its in-house,
mortgage, external-model, and ExpGuard values are **not part of focused Paper A**
and must not be cited as Paper A results. It is retained only as an archived local
exploration aid and is intentionally not linked from the paper's navigation.

## Open it

Just double-click **`index.html`** (or drag it into any browser). No server, no
build step, no dependencies — everything is inlined into the one file.

## What's inside

- **13 benchmarks** spanning 4 evaluation pools (in-distribution, in-house
  held-out, novel/OOD, domain use case) and 3 safety axes (guardrail, red-team,
  over-refusal), plus one landscape-only survey set.
- For each set: the source repo/citation, what it tests, prompt-pool
  composition (unsafe vs. safe row counts), the role it plays in the paper, and
  a plain-language caveat.
- Threshold-free **tie-aware AUPRC** (sklearn `average_precision_score`) is used
  throughout, matching the paper's corrected metric.

## Interactions

- **View** toggle — **Cards** (grid) or **Table** (a tab per benchmark).
- **Pool** / **Axis** filter chips narrow both views.
- **Cards**: click any card for a detail drawer.
- **Table**: pick a tab to see that benchmark's complete record laid out as a
  table, plus **a sample of its actual prompts** (real rows, unsafe = positive).
  Each sample has an **All / Unsafe / Safe** filter and a **search** box with a
  live count, for quick exploration.

## Files

- `index.html` — the app (self-contained; open directly).
- `samples.local.js` — optional local sampled prompts, loaded via `<script src>`
  when present. It is generated by the script below and gitignored because it can
  contain non-commercial or gated third-party text.
- `build_content_samples.py` — regenerates `samples.local.js` from the local frozen
  evaluation rows and benchmark files. Run from the repo root:
  `python3 paper-a/paper-html/explorer/build_content_samples.py` (or
  `make legacy-explorer`).
- `expguard_sample.local.json` — optional local ExpGuard input consumed by the
  builder; also gitignored.

## A note on the samples

The optional `samples.local.js` contains a small, label-balanced and
**tag-diversified** sample per
benchmark (30 rows, spread across trap-types / domains / hazards where present,
prompts truncated), not the full datasets — several sources are non-commercial
(ToxicChat, BeaverTails are CC BY-NC) and the paper reconstructs rows from the
public datasets rather than redistributing them. Each entry records the true row
count and links to the authoritative source for the complete set.

No third-party prompt text is shipped in the repository. Without the optional
local file, the archive still shows metadata but no prompt samples.
